# Лендинг марафона «Карьера»

Отдельная страница со своими стилями, от `css/` основного сайта не зависит.
Опубликована на **https://dariyap.ru/marathon/**, деплой автоматический
при пуше в `main`.

```
marathon/index.html     страница целиком: разметка и стили
marathon/apps-script.gs код для Google-таблицы (на сайте не используется)
js/marathon.js          тексты программы, FAQ, интерактив, форма
assets/marathon/        фотографии ведущих
```

Все адреса, которые нужно менять, лежат в начале `js/marathon.js`.

> **Никаких секретов в `js/marathon.js`.** Файл открыт любому посетителю
> через F12 и лежит в публичном репозитории. Токен бота, секретный ключ
> ЮKassa и пароли — только в Apps Script, на серверах Google.

## Куда падают заявки

Каналов может быть один или два, заявка уходит в оба сразу.
Достаточно, чтобы сработал хотя бы один.

### Почта — `MAIL_ENDPOINT`

Через [FormSubmit](https://formsubmit.co), регистрация не нужна. Уже настроено
на `merkulova.0032@gmail.com`.

> **Грабли:** FormSubmit активирует пару «почта + домен», а не почту вообще.
> С нового адреса сайта первая заявка не придёт — вместо неё упадёт письмо
> «Activate FormSubmit on …». Пока в нём не нажать кнопку, заявки теряются.
> Переносишь лендинг — сначала отправь пробную заявку сам и активируй.

### Таблица и Telegram — `SHEET_ENDPOINT`

Заявки и оплаты попадают в одну Google-таблицу, на листы «Заявки» и «Оплаты»,
и дублируются сообщением в Telegram.

1. Создай Google-таблицу → меню **Расширения → Apps Script**.
2. Удали содержимое редактора, вставь весь `apps-script.gs`.
3. Заполни три строки сверху:
   - `TELEGRAM_TOKEN` — от [@BotFather](https://t.me/BotFather) (`/newbot`)
   - `TELEGRAM_CHAT_ID` — от [@userinfobot](https://t.me/userinfobot),
     либо id общего чата, если уведомления нужны обеим
   - `WEBHOOK_KEY` — любая длинная строка без пробелов, придумай сама
4. **Напиши боту любое сообщение** — иначе он не имеет права писать первым.
5. Выбери функцию `setup`, нажми **Выполнить**, разреши доступ.
   Появятся оба листа, а в Telegram — «Проверка связи».
6. **Начать развёртывание → Веб-приложение**: запуск от своего имени,
   доступ — **у всех**. Скопируй ссылку `.../exec`.
7. Вставь её в `SHEET_ENDPOINT` в `js/marathon.js`.

> Поменяешь код скрипта — нужно снова развернуть **новую версию**,
> иначе Google продолжит выполнять старую.

## Оплата — `PAY_LINKS`

Человек сначала оставляет заявку (имя, телеграм, тариф), потом попадает
на оплату выбранного тарифа. Так и ник известен, и видно тех, кто до оплаты
не дошёл — им можно написать.

Ссылки создаёт Даша в кабинете ЮKassa (магазин `1339878`): конструктор
платёжных форм или «Оплата по ссылке», на 2 800 ₽ и 6 000 ₽.
Адрес возврата после оплаты — `https://dariyap.ru/marathon/`.

Секретный ключ API для этого не нужен и в код не вставляется никогда:
он даёт полный доступ к деньгам магазина.

### Уведомления об оплате в таблицу

В кабинете ЮKassa, раздел HTTP-уведомлений, указать адрес:

```
https://script.google.com/macros/s/.../exec?key=ТВОЙ_WEBHOOK_KEY
```

События — `payment.succeeded` и `refund.succeeded`.

Ключ в адресе обязателен: ЮKassa подтверждает подлинность только по своим
IP-адресам, а Apps Script IP отправителя не видит. Без ключа кто угодно,
зная адрес скрипта, записал бы в таблицу выдуманную оплату.

### Как связываются заявка и платёж

Скрипт ищет телеграм-ник в назначении платежа и, если находит, сам ставит
«оплачено» в строке заявки. Получится ли передать ник в платёж — зависит
от вида платёжной ссылки. Если нет, обе строки всё равно лежат в одной
таблице и сопоставляются по времени, сумме и почте плательщика.

## Локальный запуск

```bash
python -m http.server 8080
```

Открывать через `http://localhost:8080/marathon/`, а не файлом с диска:
из `file://` форма работать не будет.
